CSV Column Explanation
Vulnerability Name

Description: The name of the vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: Example Vulnerability Name

CVE-ID

Description: The Common Vulnerability Enumeration (CVE) Identifier for the vulnerability finding.

Field Type: String

Accepted Values: String values starting with "CVE-"

Example: CVE-2023-36054

CWE-ID

Description: The Common Weakness Enumeration (CWE) Identifier for the vulnerability finding.

Field Type: String

Accepted Values: String values starting with "CWE-"

Example: CWE-89

Description

Description: A description of the vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: Example Vulnerability Description

Severity

Description: The severity classification of the vulnerability finding.

Field Type: String

Accepted Values: Low, Medium, High, Critical

Example: High

Source

Description: The tool source that identified the vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: Snyk

Reference Name

Description: The optional reference name of any vulnerability finding references included.

Field Type: String

Accepted Values: Any String

Example: Example Vulnerability Reference Name

Reference URL

Description: The optional URL for the finding reference.

Field Type: String

Accepted Values: Any String starting with "http(s)"

Example: https://www.example.com/vulnerability_context

Reference Tags

Description: The optional Tags for the finding reference.

Field Type: String (Comma separated values)

Accepted Values: Comma separated strings

Example: security,third_party

Vulnerable File Name

Description: The name of the file related to the Vulnerability finding, if applicable.

Field Type: String

Accepted Values: Any String

Example: jquery.min.js

Sourcecode File Start Line

Description: If applicable, the start line in the file related to the Vulnerability finding.

Field Type: Integer

Accepted Values: Any Integer

Example: 1

Sourcecode File Start Column

Description: If applicable, the start column in the file related to the Vulnerability finding.

Field Type: Integer

Accepted Values: Any Integer

Example: 1

Sourcecode File End Line

Description: If applicable, the end line in the file related to the Vulnerability finding.

Field Type: Integer

Accepted Values: Any Integer

Example: 8

Sourcecode File End Column

Description: If applicable, the end column in the file related to the Vulnerability finding.

Field Type: Integer

Accepted Values: Any Integer

Example: 3

URI

Description: If applicable, the URI related to the Vulnerability finding.

Field Type: String

Accepted Values: Any String starting with "http(s)"

Example: https://www.example.com/vulnerable_endpoint

HTML Method

Description: If applicable, the HTML Method related to the Vulnerability finding.

Field Type: String

Accepted Values: Any HTML Method

Example: POST

HTML Request Parameters

Description: If applicable, the HTML request parameters related to the Vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: ?attack=yes

Attack

Description: An example or description of any attacks related to the Vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: https://www.example.com/vulnerable_endpoint?attack=yes

Evidence

Description: Any available evidence that either validates or provides context for the Vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: POST https://www.example.com/vulnerable_endpoint?attack=yes

Solution

Description: Information available to assist in remediation efforts for the Vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: Update to the latest version of the library.

Vulnerable Package

Description: If applicable, the name of the imported package or library related to the Vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: jquery

Vulnerable File Path

Description: If applicable, the file path of the file or imported package related to the Vulnerability finding.

Field Type: String

Accepted Values: Any String

Example: deb

Vulnerable Package Version

Description: If applicable, the version of the Vulnerable package or imported library.

Field Type: String

Accepted Values: Any String

Example: 2.2.2

Source Type

Description: The testing type or category for the Vulnerability finding.

Field Type: String

Accepted Values: Secret, SCA, SAST, IAC, Container, Infrastructure, DAST, DASTAPI

Example: SCA
